# Projeto

Construir uma aplicação mobile de gerenciamento de tarefas.

## Requisitos

O sistema deve permitir:

- Criar tarefas
- Editar tarefas
- Excluir tarefas
- Concluir tarefas
- Reabrir tarefas
- Categorizar tarefas
- Priorizar tarefas
- Definir data de vencimento
- Dashboard de produtividade
- Filtros
- Busca
- Persistência local

## Fluxo Obrigatório

Antes de gerar código:

1. Criar PRD.
2. Criar SDD.
3. Criar SPECS.
4. Validar Arquitetura.
5. Validar Segurança.
6. Criar Plano de Implementação.

Somente depois iniciar implementação.

## Modo Incremental

Implementar apenas uma tarefa por vez.

Exemplo:

Entrega 1:

- Theme Provider

PARAR.

Entrega 2:

- Button Component

PARAR.

Entrega 3:

- Input Component

PARAR.

Entrega 4:

- Task Schema

PARAR.

Entrega 5:

- Task Service

PARAR.

Após cada entrega:

- gerar commit sugerido;
- listar arquivos alterados;
- aguardar autorização.

## Critério de Conclusão

O projeto somente será considerado concluído quando:

- PRD concluído;
- SDD concluído;
- SPECS concluídas;
- Testes implementados;
- QA aprovado;
- Release validada.
